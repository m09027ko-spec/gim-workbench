# 仕様書（specs）

このフォルダは、Cowork が医学的妥当性・施設プロトコル・項目設計を整理し、コーディングエージェント（Claude Code / Codex / Cursor）が読むためのツール仕様書置き場です。

総診ワークベンチは、すべてのツールを `src/data/tools.ts` の `ToolDefinition` として集約する**定義駆動アーキテクチャ**を採用しています。仕様書は「どんなツールを `tools.ts` に追加するか」を決めるための入力です。

## ファイル名

- `<tool-id>.md` の形式にする
- `<tool-id>` は `src/data/tools.ts` の `id` と完全一致させる
- 例: `has-bled.md`, `af-risk.md`, `discharge-summary.md`, `doac-dose-check.md`
- ハイフン区切り、英小文字

## 使い方（標準ルート）

1. **Cowork**: `TEMPLATE.md` をコピーして `<tool-id>.md` を作る
2. **Cowork ↔ ユーザー**: 医学的妥当性・施設ルール・閾値・推奨アクションの表現を詰める
3. **ユーザー**: 「確定」へ進めることを承認する
4. **コーディングエージェント**: 仕様書を読み、`AGENTS.md` の標準手順に従って実装
5. **コーディングエージェント**: 実装完了後、ステータスを「実装済み（<エージェント識別子>, YYYY-MM-DD）」に更新

仕様書のステータスが「下書き」のまま実装着手しないでください。ユーザー確認後に「確定」へ進みます。

## ステータスの段階

| 段階 | 意味 | 次の遷移先 |
| --- | --- | --- |
| `下書き` | Cowork が作成中。ユーザー確認前 | → 確定（ユーザーがOK） |
| `確定` | ユーザー確認済み。エージェントが実装してよい | → 実装中（実装着手） |
| `実装中` | コーディングエージェントが作業中 | → 実装済み（ビルド成功・HANDOFF 更新済み） |
| `実装済み` | コード反映済み | （凍結。仕様変更時は新しい下書きで上書き） |

## 注意

- 仕様書には患者個人情報・実症例の本文を書かない
- 一般化された臨床知識・スコア定義・チェック項目・閾値・推奨アクション（業務補助表現）のみを記載
- 未確認の薬剤量・点数表・施設依存データは仕様書上で「TODO」と明示し、実装側では `todos[]` で表示する
- 既存ツールと機能が重複しそうな場合は、統合するか別ツールにするかをユーザー確認

## 既存ツールから仕様書を逆抽出する場合

`src/data/tools.ts` に既に存在するが仕様書がないツールについて仕様書を作る場合、ファイル冒頭に以下を明記してください。

> このファイルは既存実装からの逆抽出仕様書です。実装が先行しており、本仕様書は現状実装の文書化と、今後の改修方針整理のために作成されています。
