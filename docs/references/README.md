# 参考資料（references）

このフォルダは、先生（ユーザー）が集めた参考資料の置き場です。

## 重要: このフォルダは Git に上げません

`.gitignore` で `docs/references/` の中身は除外しています。**この `README.md` だけが Git に上がる**設定です。

理由:

- 論文・ガイドライン PDF などは著作権上、公開リポジトリに置けない
- 患者個人情報や施設固有情報が紛れ込むリスクを構造的に排除する
- リポジトリは「成果物だけ」というルールを徹底する

## 何を入れていいか

| 種類 | 例 | OK / NG |
| --- | --- | --- |
| スコアの原典論文 PDF | HAS-BLEDのChest 2010 | OK |
| ガイドラインの PDF や該当ページのスクリーンショット | 心不全GL、CPGなど | OK |
| 教科書・総説のメモ、抜粋 | 自分でまとめたノート | OK |
| 教育用素材（過去の研修医向けスライドの元ネタ等） | 一般化された医学知識のみ | OK |
| 院内ルール・施設固有マニュアル | 自院の感染対策マニュアル等 | **NG**（院内ストレージへ） |
| 実症例の本文・カルテ抜粋・患者情報 | - | **絶対 NG**（このフォルダ内であっても置かない） |

## 整理の目安

モジュール単位でサブフォルダを切ると、Cowork（私）が参照するときに楽です。

```text
docs/references/
├── README.md            ← これ（Git管理）
├── has-bled/            ← HAS-BLED の原典・GL（Git管理外）
│   └── Pisters2010_Chest.pdf
├── heart-failure/       ← 心不全の参考資料（Git管理外）
│   └── JCS_HF_GL.pdf
├── _general/            ← モジュールに紐づかない総合資料（Git管理外）
│   └── ...
└── ...
```

サブフォルダ名は対応するモジュールIDに揃えてください（仕様書 `docs/specs/<module-id>.md` と一致）。

## Cowork / Claude Code はどう使うか

- **Cowork（Claude Desktop）**: 仕様書を書くとき、このフォルダの資料を読んで、要約・引用・該当ページ番号を `docs/specs/<module-id>.md` の「参考文献・根拠」セクションに反映します。
- **Claude Code**: 通常は読みません。実装中に「このスコアの閾値の根拠は？」と疑問が出たときだけ、ユーザーに確認したうえで参照します。

つまり、**資料は素材、仕様書が成果物**という関係です。資料そのものを Git にコミットせず、要約と引用情報だけが仕様書経由で Git に残ります。
