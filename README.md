# 総診ワークベンチ / GIM Workbench

内科臨床で日常的に使える、医療者向けの軽量ミニツールをまとめた総合診療向けWebアプリです。計算式、スコア、薬剤確認、チェックリスト、簡易フローチャートを中心にしています。

このアプリは臨床判断を置き換えません。各ツールは補助・教育目的であり、最終判断は担当医が行ってください。施設プロトコル、添付文書、最新ガイドラインを優先します。

## 起動方法

```bash
npm install
npm run dev
```

## ビルド・テスト

```bash
npm test
npm run build
```

単一HTML版を更新する場合:

```bash
npm run build:portable
```

生成ファイルは `portable/gim-workbench.html` です。

## 医療安全上の注意

全ツールに以下を表示しています。

- 医療者向け補助ツールであり、最終判断は担当医が行う。
- 施設プロトコル・添付文書・最新ガイドラインを優先。
- 未確認データや薬剤量は仮実装せず、TODOとして明示。

患者名、患者ID、生年月日、住所、電話番号、カルテ番号、症例本文は入力・保存しない設計です。外部APIやサーバー通信は使いません。

## 未確認データ・TODO一覧

- DOAC用量チェック: 日本の添付文書、適応疾患別、腎機能別、薬剤部ルールの確認が必要。
- 3%食塩水: 詳細投与量は施設プロトコル確認が必要。
- DICスコア: ISTH DICまたは日本救急医学会DICなど、施設採用基準選択後に点数表を実装。
- 抗菌薬腎機能用量: `src/data/antibioticRenalDosing.ts` にTODOのみを保持。用量は未実装。
- 抗菌薬内服スイッチ: 候補提示の枠のみ。感染臓器別候補薬は施設ルールで確認。
- ワルファリン: 開始量は断定せず、注意因子表示に留める。
- インスリン補正スケール: 教育用テンプレートのみ。実運用前に施設スケールへ置換。

## 今後追加すべき根拠資料

- 各薬剤の日本語添付文書
- 施設薬剤部の腎機能別投与量表
- 施設の3%食塩水、DIC、インスリン補正スケール、ワルファリン導入プロトコル
- 肺炎、敗血症、輸血、抗菌薬適正使用に関する院内ルール

資料そのものは `docs/references/` にローカル保存し、Gitには原則コミットしません。

## 施設プロトコルへの置換方法

1. 施設で確認済みの資料を `docs/references/` に置く。
2. 必要に応じて `docs/specs/<tool-id>.md` に確認済み内容を要約する。
3. 薬剤量テーブルは `src/data/*.ts` に分離して追加する。
4. 未確認の数値はコードに入れず、TODO表示のまま残す。
5. `npm test` と `npm run build` を通す。
